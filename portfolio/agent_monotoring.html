Infraestructura Automatizada con Bash, Docker, Ansible y Oracle Cloud

Introducción

En este proyecto desarrollé una solución integral de monitoreo para servidores y máquinas virtuales. El sistema extrae métricas de salud en tiempo real de múltiples nodos y las centraliza en un dashboard interactivo, eliminando la necesidad de supervisión manual individual.

El Problema

Monitorear el estado de salud (CPU, RAM, Disco) de múltiples servidores de forma manual es ineficiente y propenso a errores. El objetivo fue crear una arquitectura que permitiera observar toda la infraestructura desde un solo punto, garantizando escalabilidad, portabilidad y bajo impacto en el rendimiento de los nodos esclavos.

Arquitectura del Proyecto

La solución está modularizada y estructurada en cuatro capas principales:

Agente de Extracción

Script optimizado en Bash que recolecta métricas vivas del sistema nativo y las empaqueta en JSON.

Backend API

API REST construida en Python con Flask para procesar, validar e inyectar de forma segura los flujos de datos.

Persistencia

Base de datos relacional PostgreSQL encargada del almacenamiento estructurado e histórico de las métricas.

Frontend

Dashboard reactivo con tarjetas dinámicas que exponen visualmente y en tiempo real el estado de cada nodo.

Tecnologías Utilizadas

Python Flask PostgreSQL Docker Docker Compose Ansible Bash Scripting Oracle Cloud (OCI)

Infraestructura en la Nube (Oracle Cloud)

A diferencia de un entorno local tradicional, todo el core central del sistema fue desplegado sobre una arquitectura Cloud en una instancia de **Oracle Cloud Infrastructure (OCI)**. Esto implicó la gestión avanzada de seguridad y redes:

  • Virtual Cloud Network (VCN): Orquestación de listas de seguridad y reglas de entrada (Ingress Rules) granulares para restringir y permitir el tráfico de métricas bajo protocolos específicos.
  • Seguridad de Sistema (Hardening): Cierre de accesos vulnerables, implementación de autenticación estricta vía llaves criptográficas SSH RSA de 4096 bits y ofuscación de puertos por defecto.
  • Disponibilidad Continua: Operatividad 24/7 en la nube, garantizando que agentes distribuidos en cualquier parte del mundo reporten sus payloads sin latencia.

Automatización con Ansible (IaC)

Para mitigar el error humano y acelerar los despliegues masivos en nuevos nodos o VMs, utilicé **Ansible**. Diseñé Playbooks automatizados que preparan el entorno de la máquina remota, inyectan el script del agente y configuran un servicio de ejecución en segundo plano administrado de forma nativa por **Systemd**.

Contenerización con Docker Compose

Con la finalidad de garantizar la portabilidad absoluta del servidor central en Oracle Cloud, encapsulé los entornos independientes de la API de Flask y la persistencia de datos. Utilizar **Docker Compose** asegura un entorno inmutable e idéntico tanto en la fase de laboratorio local como en producción.

El Agente de Monitoreo (Bash)

El script principal aprovecha comandos de bajo nivel del kernel de Linux para computar con precisión milimétrica la carga de recursos de hardware y realiza una petición HTTP POST segura hacia la API central de forma automatizada cada 10 segundos.

Conclusión y Próximos Pasos

Este proyecto demuestra la potencia técnica de combinar la Infraestructura como Código (IaC) con la portabilidad de los microservicios en contenedores. Se logró reducir de manera sustancial los tiempos de configuración y aprovisionamiento de nuevos nodos.

Siguiente paso estratégico: Implementar un pipeline de CI/CD automatizado mediante Jenkins para agilizar el ciclo de despliegue de las actualizaciones del núcleo del dashboard.
